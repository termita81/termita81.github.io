# Javascript style

1. Do not include semicolons at the end of Javascript statement, unless necessary
2. Avoid un-necessary comments, that state the obvious. Prefer "why" comments, rather than "what" or "how", unless the code is complicated
3. Write logical code, that is easy to follow, using approprite abstractions
4. Avoid functions that do nothing but call other functions

# Project structure

Files under docs directory are build results/outputs, no direct change to be done there

# Work style

1. Create feature branches for each body of work
2. Append your info ("With AI: <model name and version>") at the end of the commit messages
